NAnt macrodef Help Task Reference <macrodef> v0.1.0

<macrodef>

[This is preliminary documentation and subject to change.]

Defines a new task.

Derived from Ant's macrodef task. Defines a new task, called name, which uses the sequential element as a template. The new task can have xml attributes and xml child elements.

Parameters

AttributeTypeDescriptionRequired
namestring The name of the macro. True
failonerrorboolFalse
ifboolFalse
unlessboolFalse
verboseboolFalse

Nested Elements:

<sequential>

The tasks to execute when this macro is invoked.

Contains the template for the macro - the tasks that should be executed when the macro is called.

</sequential>

<attributes>

Attributes to the task - simple xml attributes on the macro invocation.
<attribute>

Describes a parameter/attribute of the macro. Can be accessed as properties within the macro definition (i.e. ${prop}).

Parameters

AttributeTypeDescriptionRequired
defaultstring Default value - the property will be set to this if the attribute is not present. False
namestring The name of the attribute. False
propertystring Property name to store the value in - defaults to the name of the attribute. False
</attribute>

</attributes>

<elements>

Attributes to the task - xml child elements of the macro invocation.
<element>

Describe nested elements that can be supplied to the macrodef. These elements are callable by name.

Parameters

AttributeTypeDescriptionRequired
namestring The name of the element. False
</element>

</elements>

Examples

  • Simple Macro.

                <macrodef name="mytask">
                    <sequential>
                        <echo messasge="mytask invoked!"/>
                    </sequential>
            </macrodef>
            <mytask/>

  • Receive Parameters.

                <macrodef name="assert-equals">
            <attributes>
            <attribute name="name"/>
            <attribute name="expected"/>
            <attribute name="actual"/>
            </attributes>
                  <sequential>
            <fail if="${ expected != actual}" message="${name}: expected '${expected}' but was '${actual}'"/>
            </sequential>
            </macrodef>

  • Receive Callable Elements.

    <macrodef name="macro-with-elements">
                    <elements>
                        <element name="element1"/>
                    </elements>
                    <sequential>
                        <echo message="before element1"/>
                        <element1/>
                        <echo message="after element1"/>
                    </sequential>
                </macrodef>
                <macro-with-elements>
                    <element1>
                        <echo message="element1"/>
                    </element1>
                </macro-with-elements>
